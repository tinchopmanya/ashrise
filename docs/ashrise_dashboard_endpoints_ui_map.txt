ASHRISE DASHBOARD — MAPA DE ENDPOINTS, CAMPOS Y USO EN UI (v2)
Fecha: 2026-04-21

CAMBIOS VS v1
- Se agrega la entidad `tasks` con FK a ideas/projects/candidates
- Se agrega endpoint de grafo para Project Detail
- Se consolida Ideas + Tasks en un único módulo con 3 subvistas
- Se agrega endpoint de workspace que junta idea + tasks + relacionadas

NOTACIÓN
- CURRENT   = endpoints ya existentes o esperables por el core actual
- PROPOSED  = endpoints recomendados para el dashboard
- NEW       = entidad/endpoint nuevo que requiere migración
- LANGFUSE  = datos que vienen por integración, no duplicados en Ashrise

3. IDEAS & TASKS  [MÓDULO EXPANDIDO]
====================================================================

Este módulo fusiona el antiguo "Ideas" con la nueva entidad `tasks`.
Tiene 3 subvistas: árbol, split-arrows, board kanban.

CURRENT
- GET  /ideas?status=
- POST /ideas
- PATCH /ideas/{id}

IDEA FIELDS
  id, project_id, raw_text, title, description, tags[],
  source, source_ref, status, promoted_to, triage_notes,
  parent_id, cross_links[], created_at, triaged_at

NUEVA ENTIDAD: TASKS  [NEW]
- GET    /tasks?idea_id=&project_id=&candidate_id=&status=
- GET    /tasks/{id}
- POST   /tasks
- PATCH  /tasks/{id}
- DELETE /tasks/{id}
- POST   /tasks/{id}/promote

TASK FIELDS
  id, idea_id, project_id, candidate_id,
  title, description,
  status  ('backlog','ready','progress','blocked','done'),
  priority, position, tags[],
  promoted_to, created_at, updated_at, closed_at

PROPOSED AGGREGATED
- GET /dashboard/ideas/overview
    counts por status, por source
    lista de ideas con task_counts embebidos
    cross_links detectados

- GET /dashboard/ideas/{id}/workspace   [NEW]
    idea + tasks[] + sibling_ideas_in_same_tag + suggested_next

- GET /dashboard/tasks/board
    all tasks grouped by status, filtrable por idea o project
    devuelve posición ordenada para drag&drop

DISPLAY IN HTML

Subvista 1: "Árbol"
- sidebar izquierda: árbol de ideas con expand/collapse
- click en idea → panel central con descripción, tags, turns de conversación
- chips de hashtags con color derivado del tag
- composer inline para agregar turn (claude/codex/gemini/yo)

Subvista 2: "Tareas" (split-arrows)
- izquierda: círculo grande con la idea raíz y su contador de tareas
- derecha: flechas numeradas con gradiente (cian→violeta→rosa) según index
- cada flecha tiene título, descripción, status chip, edit inline, delete
- al crear, entra al backlog del board
- conector curvo del círculo a cada flecha

Subvista 3: "Tablero" (kanban)
- columnas: Backlog, Ready, In progress, Blocked, Done
- drag&drop con feedback de columna activa
- filtro por idea
- contador por columna
- color del borde izquierdo de la card según idea de origen

12. EDITING MODE
====================================================================

La versión final soporta edición de:
- project metadata
- project_state
- handoffs (create + resolve)
- decisions (create + supersede)
- ideas triage
- candidate metadata
- candidate promotion
- queue re-enqueue
- run-agent triggers
- tasks CRUD completo

PATRONES UI
- side drawers para creación/edición
- inline edits para campos simples (title, next_step)
- confirmación modal para destructivo (delete task, supersede decision)
- audit trail visible post-save (quién, cuándo, qué cambió)
